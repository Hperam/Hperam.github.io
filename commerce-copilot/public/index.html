Multimodal commerce intelligence

Product discovery that starts with grounded catalog retrieval then layers explainable AI reasoning on top.

Commerce Copilot is a client-first portfolio project for multimodal shopping discovery. Users can search by text, voice, or image, review ranked catalog candidates, and optionally unlock structured recommendations with their own OpenAI API key for the current browser session only.

Input → Retrieval → Grounded candidates → AI reasoning → Share summary

Architecture

Static-hosted, grounded, and session-scoped by design

Input

Text, voice transcript, and image upload are captured in-browser.

→
Retrieval

Catalog candidates are filtered and ranked locally using query, tags, and budget signals.

→
AI reasoning

Only the top grounded candidates and user intent are sent to OpenAI after the user enables a session key.

→
Output

Structured picks, tradeoffs, confidence notes, and a share summary are rendered in the client.

Hosting model

Built for GitHub Pages static export. No required backend for the demo path, which keeps the experience easy to inspect and deploy.

Grounding model

Retrieval happens before recommendation. The AI layer is constrained to reason only over ranked catalog candidates instead of inventing products.

Session security

The OpenAI key is user-supplied, stored only in session scope for the current tab, and removable at any time with a clear-session control.

Portfolio signal

The demo showcases full-stack product thinking: interaction design, retrieval logic, AI orchestration, explainability, and honest UX around security limits.
