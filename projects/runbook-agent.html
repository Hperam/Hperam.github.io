Project brief + implementation plan

Runbook Agent

An incident response copilot that ingests logs, metrics, traces, and service docs to produce grounded triage plans, suggested root causes, and safe next steps for engineering teams running cloud services.

Why now: agentic DevOps is hot, but trust still matters Best signal: backend systems + AI + observability Status: architecture scoped for public build

What this project proves

  • You can design systems that combine AI with production-grade engineering guardrails.
  • You understand observability signals, failure modes, and operational workflows.
  • You can build useful AI products without pretending the model should take unsafe actions automatically.

Core experience

An engineer opens an incident, the system pulls recent logs and metrics, ranks likely services involved, retrieves runbooks and design docs, and generates a structured triage summary with evidence, uncertainty, and suggested next checks. Every recommendation is linked back to source telemetry or internal docs.

Architecture outline

  • Telemetry ingest layer pulling CloudWatch, Prometheus, or OpenTelemetry traces into a normalized event timeline.
  • Document ingestion pipeline for runbooks, oncall guides, service ownership docs, and historical postmortems.
  • Retrieval and ranking layer combining vector search with incident-specific signal scoring.
  • LLM reasoning layer that generates a triage report with citations, confidence, and blocked assumptions.
  • Approval-gated action center for low-risk follow-up steps like querying additional logs or opening Jira tickets.

Suggested stack

  • Frontend: Next.js, TypeScript, Tailwind or custom CSS
  • Backend: Python FastAPI for orchestration
  • AI: OpenAI Responses API, embeddings, eval harness
  • Storage: Postgres, Redis, vector store
  • Infra: AWS ECS or Lambda, CloudWatch, S3

Hiring-team signal

This reads well for backend, platform, SRE-adjacent, and infrastructure-heavy SDE 2 roles because it blends system design, observability, and practical AI product judgment.
