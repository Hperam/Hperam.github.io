Multimodal Commerce Copilot

A product discovery assistant that accepts screenshots, product photos, voice prompts, and typed questions to compare items, explain tradeoffs, and produce grounded recommendations backed by structured catalog data.

Why now: multimodal interfaces are finally expected Best signal: polished UX + full-stack AI product execution Status: source code now included in repo

What this project proves

  • You can build an AI product that feels consumer-grade, not just technically correct.
  • You can ground model output in structured product data and retrieval rather than hallucinated recommendations.
  • You can tie together frontend polish, backend APIs, search, and AI orchestration in one coherent system.

Core experience

A user uploads an image of a product or asks a voice question like “find me a lighter version of this under $200.” The system extracts attributes, matches against catalog data, explains the tradeoffs between candidate items, and generates a clean summary that can be shared with friends or copied into a purchase list.

What is implemented

  • Standalone Node and Express app with a polished browser UI.
  • Typed shopping requests, optional browser voice capture, and optional image upload.
  • Catalog-grounded retrieval before model reasoning so recommendations stay anchored to real products.
  • Structured recommendation output with reasons, tradeoffs, follow-up prompts, and a share card.
  • Fallback mode when no OpenAI API key is configured, so the project still runs locally out of the box.

Architecture outline

  • Catalog ingestion pipeline that normalizes product specs, reviews, pricing, and image embeddings.
  • Multimodal input layer for image upload, speech-to-text, and conversational follow-up.
  • Retrieval and ranking stack combining keyword filters, semantic similarity, and price or feature constraints.
  • Response generation layer that cites grounded attributes and explains recommendation tradeoffs clearly.
  • Shareable result pages that feel like a real product experience instead of an internal demo.

Actual stack

  • Frontend: vanilla HTML, CSS, and JavaScript
  • Backend: Node.js and Express
  • AI: OpenAI Responses API with structured outputs and image input
  • Data: local curated catalog with retrieval ranking
  • Developer setup: npm, dotenv, local runtime docs
